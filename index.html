The Problem

Claude Code wasn't built for multiple accounts.

‚öô
Manual config swaps
Editing ~/.claude/credentials every time you switch contexts
‚óé
Orphaned sessions
Stale project data silently eating your disk space
‚óĽ
No visibility
No way to see which account is active or how much space Claude uses

CCM fixes all of this with a single command.

What's Inside
‚ü©_
Account Switching
ccm switch <id>
Rotate between accounts in one command. Aliases, history, undo, reorder, import/export.
‚äě
Project Bindings
ccm bind . work
Bind project directories to accounts. Running ccm switch in a bound directory auto-switches to the right account.
‚ä°
Session Management
ccm session search
List, inspect, relocate, search, and clean sessions. Full-text search across all conversation history.
‚äü
Usage Analytics
ccm usage history
Token usage breakdown by project and day. Disk usage, session counts, project ranking.
‚äē
Health & Maintenance
ccm doctor
13 health checks: disk size, tmp bloat, orphaned processes, stale locks, hook config. Auto-fix with --fix. Clean tmp files, kill leaked processes.
‚äė
Token Optimization
ccm optimize
Analyze context window footprint. Flag unused plugins and oversized CLAUDE.md files inflating token usage.
‚äó
Launch Modes
ccm launch auto
Launch Claude Code with auto, yolo, plan, or safe mode. Terminal reset on exit fixes broken Ctrl-C in tmux.
‚äô
Project Init
ccm init
Auto-generate .claudeignore by project type. Saves 40-60% tokens by excluding node_modules, build dirs, and more.
‚äõ
Permissions Audit
ccm permissions audit
Find duplicate, contradictory, and dead "Always Allow" rules bloating your settings.json. Auto-fix with --fix.
‚äú
Environment Snapshots
ccm env snapshot
Save and restore your Claude Code config. Audit MCP servers for token waste.
